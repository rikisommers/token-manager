## Integration Pattern Research

The package `@alliedtelesis-labs-nz/atui-components-stencil` is installed at version 0.1.174 in the Yarn workspace (hoisted to root node_modules).

Package structure:
- `loader/index.js` — exports `defineCustomElements(window?)` which registers all custom elements globally
- `loader/index.d.ts` — typed as `defineCustomElements(win?: Window, opts?): void`
- `dist/react/components.ts` — React output target bindings (requires `@stencil/react-output-target/runtime` which is NOT installed — do not use this approach)
- `styles.css` (root) — `@import './dist/atui-components-stencil/atui-components-stencil.css'` (cannot be imported directly in Next.js CSS from node_modules without config)
- `dist/atui-components-stencil/atui-components-stencil.css` — the actual CSS file

**Correct integration strategy for Next.js 13.5.6 App Router:**
1. Create a `'use client'` component (`AtuiDevTest.tsx`)
2. In `useEffect`, dynamically import `defineCustomElements` from `@alliedtelesis-labs-nz/atui-components-stencil/loader` and call it — this avoids SSR issues (Stencil references `window` which is undefined in Node)
3. Import the CSS directly: `import '@alliedtelesis-labs-nz/atui-components-stencil/dist/atui-components-stencil/atui-components-stencil.css'`
4. Use the `<at-button>` JSX tag directly in the render (TypeScript needs a JSX namespace declaration to avoid TS errors)
5. The route page `src/app/dev-test/page.tsx` is a simple server component that renders `<AtuiDevTest />`

**TypeScript JSX note:** Custom HTML elements not in the React JSX namespace cause TS errors. Add a `declare global { namespace JSX { interface IntrinsicElements { 'at-button': React.HTMLAttributes<HTMLElement> & { variant?: string; disabled?: boolean } } } }` declaration in AtuiDevTest.tsx to satisfy TypeScript without installing additional type packages.

**next.config.js note:** Next.js by default allows CSS imports from node_modules inside components marked `'use client'`. No additional webpack config should be needed for CSS import from node_modules in Next.js 13.5.6.

**File 1: `src/components/AtuiDevTest.tsx`**

This is a 'use client' React component that:
1. Declares `'use client'` at the top
2. Imports React and useEffect from 'react'
3. Imports the ATUI CSS: `import '@alliedtelesis-labs-nz/atui-components-stencil/dist/atui-components-stencil/atui-components-stencil.css'`
4. Adds a JSX global namespace declaration for `at-button` so TypeScript does not error on the unknown element:
   ```typescript
   declare global {
     namespace JSX {
       interface IntrinsicElements {
         'at-button': React.HTMLAttributes<HTMLElement> & {
           variant?: string;
           disabled?: boolean;
           size?: string;
         };
       }
     }
   }
   ```
5. In `useEffect` (runs only on client), dynamically imports `defineCustomElements` from `@alliedtelesis-labs-nz/atui-components-stencil/loader` and calls it:
   ```typescript
   useEffect(() => {
     import('@alliedtelesis-labs-nz/atui-components-stencil/loader').then(({ defineCustomElements }) => {
       defineCustomElements(window);
     });
   }, []);
   ```
