## Milestone Summary

**Key Decisions:**
- shadcn/ui components manually created (CLI requires interactive input unavailable in Claude Code); used canonical shadcn/ui source directly
- Tab switcher state preserved in React state; shadcn TabsList used as UI-only (not TabsContent) — preserves form state across tab switches
- ATUI Stencil: `next/dynamic` + `ssr:false` + `useEffect defineCustomElements` — avoids hydration mismatch in App Router
- `LayoutShell` as 'use client' component — keeps root layout.tsx a clean server component (required for Next.js metadata export)
- `CollectionLayoutClient` fetches collection name client-side — keeps collection layout.tsx as server component
- `pathname.startsWith('/collections')` in LayoutShell — suppresses root sidebar for all collection-scoped routes
- localStorage key `atui-selected-collection-id` used by AppSidebar for collection persistence
- `didMountRef` pattern in Settings page — prevents auto-save from firing during initial data load

**Issues Resolved:**
- Pre-existing TypeScript error in `token.service.ts` (string index on `{}`) resolved in 04-06 — yarn build now passes cleanly
- `ui.utils.ts` TS2339 auto-fixed via Record<string, unknown> cast

**Technical Debt:**
- `/dev-test` sandbox page remains in production build (ATUI integration test artifact)
- View Tokens section preserved as hidden div in Tokens page — code intact but not navigable via sidebar
