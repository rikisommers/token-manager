**Fix 1 — export route auth header:**
In `src/app/api/export/figma/route.ts`, change the Figma API call headers from:
```typescript
'Authorization': `Bearer ${figmaToken}`,
```
to:
```typescript
'X-Figma-Token': figmaToken,
```
This is the correct Figma REST API authentication header. Also update the POST body to accept `collectionId` (optional) for targeting a specific variable collection — the existing `variableCollectionId: 'default'` placeholder should become `variableCollectionId: collectionId || 'default'` in `transformToFigmaVariables`.

**Fix 2 — extend sourceMetadata types:**
In `src/types/collection.types.ts`, extend `ISourceMetadata`:
```typescript
export interface ISourceMetadata {
  // Existing GitHub fields
  repo: string | null;
  branch: string | null;
  path: string | null;
  // New Figma fields
  type: 'github' | 'figma' | null;   // discriminator; null = no upstream
  figmaFileKey: string | null;
  figmaCollectionId: string | null;  // variable collection ID in Figma
}
```
The `type` field acts as a discriminator so the source context bar knows which icon to show.

**Fix 3 — update Mongoose schema:**
In `src/lib/db/models/TokenCollection.ts`, add the new fields to `sourceMetadataSchema`:
```typescript
const sourceMetadataSchema = new Schema(
  {
    repo:               { type: String, default: null },
    branch:             { type: String, default: null },
    path:               { type: String, default: null },
    type:               { type: String, enum: ['github', 'figma', null], default: null },
    figmaFileKey:       { type: String, default: null },
    figmaCollectionId:  { type: String, default: null },
  },
  { _id: false }
);
```
Existing documents without these fields will have them as null (Mongoose defaults), so no migration needed.
