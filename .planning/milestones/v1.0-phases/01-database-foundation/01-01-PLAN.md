<task type="auto">
  <name>Task 2: Create MongoDB singleton connection module</name>
  <files>src/lib/mongodb.ts</files>
  <action>
    Create `src/lib/mongodb.ts` implementing the Next.js mongoose singleton pattern. The module must:

    1. Read `MONGODB_URI` from `process.env.MONGODB_URI`.
    2. Throw a clear error at module load time if `MONGODB_URI` is undefined: `throw new Error('MONGODB_URI environment variable is not set. Copy .env.local.example to .env.local and set your MongoDB connection string.')`.
    3. Use a global cache (`global.__mongoose_cache`) to store the connection promise across hot-reloads in Next.js dev mode — this is the standard singleton pattern for Next.js 13.
    4. Export a default async function `dbConnect()` that returns `mongoose.connect(uri, opts)` reusing the cached promise if it exists.
    5. Set mongoose connection options: `{ bufferCommands: false }` — disables query buffering so failed connections surface immediately as errors rather than hanging.
    6. On the mongoose connection object, register event handlers:
       - `connected`: `console.log('[MongoDB] Connected')`
       - `error`: `console.error('[MongoDB] Connection error:', err)`
       - `disconnected`: `console.warn('[MongoDB] Disconnected')`

    Full pattern (adapt as needed):
    ```typescript
    import mongoose from 'mongoose';

    const MONGODB_URI = process.env.MONGODB_URI;

    if (!MONGODB_URI) {
      throw new Error(
        'MONGODB_URI environment variable is not set. Copy .env.local.example to .env.local and set your MongoDB connection string.'
      );
    }

    // Extend global type for caching
    declare global {
      var __mongoose_cache: { conn: typeof mongoose | null; promise: Promise<typeof mongoose> | null };
    }

    let cached = global.__mongoose_cache ?? { conn: null, promise: null };
    global.__mongoose_cache = cached;

    async function dbConnect(): Promise<typeof mongoose> {
      if (cached.conn) return cached.conn;

      if (!cached.promise) {
        cached.promise = mongoose
          .connect(MONGODB_URI!, { bufferCommands: false })
          .then((mong) => {
            console.log('[MongoDB] Connected');
            return mong;
          });
      }

      cached.conn = await cached.promise;
      return cached.conn;
    }

    export default dbConnect;
    ```

    Note: Do NOT call `dbConnect()` at module load time. API routes will call it explicitly before querying. This keeps the module importable even without a live MongoDB connection (important for build time).
