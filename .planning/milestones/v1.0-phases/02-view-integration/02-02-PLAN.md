Rewrite `src/app/page.tsx` to integrate the CollectionSelector and MongoDB token fetching. The file is a full client component (`'use client'`).

**New state:**
```typescript
const [collections, setCollections] = useState<{ _id: string; name: string }[]>([]);
const [selectedId, setSelectedId] = useState<string>('local');
const [tableLoading, setTableLoading] = useState(false);   // spinner overlay on table
const [toast, setToast] = useState<{ message: string; type: 'success' | 'error' | 'info' } | null>(null);
```
Keep existing `tokenData`, `loading` (initial full-page load), `error` state.

**AbortController ref:**
```typescript
const abortControllerRef = useRef<AbortController | null>(null);
```

**On mount — two effects:**

1. `useEffect` to load the collections list from `/api/collections`:
   - On success: `setCollections(data.collections)`; then read `localStorage.getItem('atui-selected-collection-id')`; if the stored id exists in the returned collections array, call `handleSelectionChange(storedId)`, else `handleSelectionChange('local')`
   - On error: show error toast "Failed to load collections list" and show only Local Files (collections stays empty)

2. Keep the existing `useEffect` that calls `fetchTokens()` for the initial Local Files load — do not remove it.

**`handleSelectionChange(id: string)` function:**
- Cancel any in-flight request: `abortControllerRef.current?.abort(); abortControllerRef.current = new AbortController()`
- `setSelectedId(id)`
- `localStorage.setItem('atui-selected-collection-id', id)`
- If `id === 'local'`: call `fetchTokens()` (existing function, no changes needed) and return
- If `id` is a MongoDB ObjectId: `setTableLoading(true)`, then `fetch('/api/collections/' + id, { signal: abortControllerRef.current.signal })`
  - On success: parse `data.collection.tokens`, transform to the `Record<string, TokenGroup[]>` shape that `TokenTable` expects (see note below), `setTokenData(transformed)`
  - On abort error (`err.name === 'AbortError'`): do nothing (user switched away)
  - On other error: `setToast({ message: 'Failed to load collection', type: 'error' })`, `setTokenData({})` (clear table)
  - Finally: `setTableLoading(false)`

**Token data transformation for MongoDB collections:**
The MongoDB `tokens` field is the raw W3C Design Token JSON object (e.g., `{ "color-base": { "blue": { "$value": "#0000ff", "$type": "color" } } }`). The TokenTable component expects `Record<string, TokenGroup[]>` where each key is a section name and each value is an array of `{ path, token: { value, type }, filePath, section }` objects.

To produce this shape from a MongoDB collection's tokens, reuse the same transformation logic from `TokenUpdater` (filesystem-based) or replicate the flat mapping inline:
- Top-level keys of the tokens object → section names
- Recursively walk each section's subtree; when a node has a `$value` key, it is a token leaf: `{ path: dotted.key.path, token: { value: node.$value, type: node.$type || 'other' }, filePath: collectionName, section: topLevelKey }`
- Do NOT call `updater.getAllTokensWithResolvedRefs()` for MongoDB tokens — that reads from the filesystem

Create a local helper function `flattenMongoTokens(tokens: Record<string, unknown>, collectionName: string): Record<string, TokenGroup[]>` in `page.tsx` (not a new file).

**Spinner overlay:**
