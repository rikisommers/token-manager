Create `src/app/api/collections/route.ts` as a Next.js App Router route handler.

The GET handler must:
1. Call `await dbConnect()` (import from `@/lib/mongodb`) before any queries
2. Query `TokenCollection.find({}, { name: 1, createdAt: 1 }).lean()` — project only `name` and `createdAt`; `.lean()` returns plain objects (not Mongoose Documents), which are directly JSON-serialisable
3. Map results to strip Mongoose internal fields: return `_id` as a string (`.toString()` since lean() keeps it as ObjectId), `name` as string, `createdAt` as ISO string
4. Return `NextResponse.json({ collections: [...] })` with status 200
5. On error: catch, `console.error`, return `NextResponse.json({ error: 'Failed to fetch collections' }, { status: 500 })`

Response shape (matches ANGULAR_PARITY.md contract):
```typescript
{ collections: Array<{ _id: string; name: string; createdAt: string }> }
```

Import pattern (matching established codebase convention):
```typescript
import { NextResponse } from 'next/server';
import dbConnect from '@/lib/mongodb';
import TokenCollection from '@/lib/db/models/TokenCollection';
```

Do NOT sort results in the query — let MongoDB return them in natural order. The UI selector will display them as received.

Do NOT import from `@/types/collection.types.ts` in the route — the lean projection already defines the shape; inline the response type or use `as`.
