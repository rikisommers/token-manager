  <how-to-verify>
Visit http://localhost:3000 and run through each scenario.

**Pre-condition:** Have at least one MongoDB collection saved (use the Generate page to save one if needed).

**Scenario 1 — Action buttons visibility:**
1. Select "Local Files" from the collection selector.
2. Expected: No action buttons are visible below the selector.
3. Select any MongoDB collection.
4. Expected: Three buttons appear below the selector: "🗑️ Delete", "✏️ Rename", "📋 Duplicate".

**Scenario 2 — Rename collection:**
1. With a MongoDB collection selected, click "✏️ Rename".
2. Expected: Modal opens pre-filled with the current collection name.
3. Clear the input and type an existing collection's name (if you have multiple).
4. Expected: Inline error "A collection named '...' already exists." appears and Save button is disabled.
5. Clear to a new unique name (e.g. "renamed-collection") and click Save.
6. Expected: Modal closes, selector immediately shows "renamed-collection", toast "Renamed to 'renamed-collection'" appears.

**Scenario 3 — Duplicate collection:**
1. Select a collection and click "📋 Duplicate".
2. Expected: Modal opens pre-filled with "Copy of [original name]".
3. Change the name or accept the default, click Duplicate.
4. Expected: Modal closes, selector updates with the new copy and automatically selects it, toast "Duplicated as '[new name]'" appears.
5. Verify the duplicate is visible in the selector list.

**Scenario 4 — Delete collection:**
1. Select any MongoDB collection (the duplicate from Scenario 3 is ideal — keeps original data intact).
2. Click "🗑️ Delete".
3. Expected: Confirmation modal opens: "Delete '[name]'? This cannot be undone." with Cancel and a red Delete button.
4. Click Cancel. Expected: Modal closes, collection remains selected and in list.
5. Click "🗑️ Delete" again, then click Delete (red button).
6. Expected: Modal closes, selector reverts to "Local Files", deleted collection is gone from the list, toast "Collection deleted" appears, page shows local file tokens.

**Scenario 5 — Rename duplicate-name guard:**
1. With two or more MongoDB collections, select one and click "✏️ Rename".
2. Type the name of a different existing collection.
3. Expected: Inline error appears and Save is disabled — cannot save.
4. Click Cancel. No change made.

**Scenario 6 — Persistence check:**
1. Rename a collection, then navigate away and back (or refresh the page).
2. Expected: Collection appears with its new name in the selector.
