    The route follows the EXACT same whole-array $set pattern as `PUT /api/collections/[id]/themes/[themeId]/route.ts`:
    1. `await dbConnect()`
    2. `TokenCollection.findById(params.id).lean() as Record<string, unknown> | null`
    3. Cast themes array: `(collection.themes as Array<Record<string, unknown>>) ?? []`
    4. `themes.findIndex((t) => (t.id as string) === params.themeId)` — return 404 if -1
    5. Build `updatedTheme = { ...themes[themeIndex], tokens: body.tokens }`
    6. Build `updatedThemes` array (slice/spread pattern)
    7. `TokenCollection.findByIdAndUpdate(params.id, { $set: { themes: updatedThemes } }).lean()`
    8. Return `NextResponse.json({ tokens: body.tokens })`

    **Source-group guard (SUCCESS CRITERION in roadmap):**
    After finding the theme, extract `const groups = (theme.groups as Record<string, string>) ?? {}`.
    Check each root-level group in `body.tokens`:
    ```typescript
    const hasSourceWrite = (body.tokens as TokenGroup[]).some(g => groups[g.id] === 'source');
    if (hasSourceWrite) {
      return NextResponse.json({ error: 'Cannot write to source groups' }, { status: 422 });
    }
    ```
    This check is on root groups only (the top-level TokenGroup nodes in the tree), not recursively on children — group state in `theme.groups` uses root-level group IDs.

    **Validation:**
    - `!Array.isArray(body.tokens)` → 400 with `{ error: 'tokens must be an array' }`
    - Collection not found → 404 with `{ error: 'Collection or theme not found' }`
    - Theme index -1 → 404 (same message)

    **Imports:**
    ```typescript
    import { NextResponse } from 'next/server';
    import dbConnect from '@/lib/mongodb';
    import TokenCollection from '@/lib/db/models/TokenCollection';
    import type { TokenGroup } from '@/types';
    ```

    Do NOT import ITheme — the route works with raw Mongoose lean objects cast through unknown.

    **Error handling:** Wrap all logic in try/catch; log `'[PATCH /api/collections/[id]/themes/[themeId]/tokens]'`; return 500 on catch.
