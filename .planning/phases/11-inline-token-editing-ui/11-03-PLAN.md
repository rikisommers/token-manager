    Wire `TokenGeneratorForm` in `tokens/page.tsx` to use `activeThemeTokens` as the token data source when a named theme is active and the selected group is Enabled. Source groups get `isReadOnly`. Reset button wired via callback.

    **1. Derive active group state:**
    Add a `useMemo` that computes the current selected group's state in the active theme:
    ```typescript
    const activeGroupState = useMemo<'enabled' | 'source' | 'disabled' | null>(() => {
      if (!activeThemeId || !selectedGroupId) return null;
      const theme = themes.find(t => t.id === activeThemeId);
      if (!theme) return null;
      return (theme.groups[selectedGroupId] ?? 'disabled') as 'enabled' | 'source' | 'disabled';
    }, [activeThemeId, selectedGroupId, themes]);
    ```

    **2. Compute isReadOnly for the form:**
    ```typescript
    const isThemeReadOnly = activeGroupState === 'source';
    ```

    **3. Add findTokenMasterValue helper** (inline in page.tsx, not a separate utility):
    ```typescript
    // Find the master collection value for a token by groupId + tokenPath
    const findMasterValue = useCallback((groupId: string, tokenPath: string): string | undefined => {
      const group = findGroupById(masterGroups, groupId);
      const token = group?.tokens.find(t => t.path === tokenPath);
      return token !== undefined ? String(token.value ?? '') : undefined;
    }, [masterGroups]);
    ```
    Import `findGroupById` from `@/utils` if not already imported.

    **4. Add onResetToDefault handler:**
    ```typescript
    const handleResetToDefault = useCallback((groupId: string, tokenId: string, masterValue: string) => {
      if (!activeThemeId) return;
      // Update the local activeThemeTokens copy
      const updatedTokens = activeThemeTokens.map(g =>
        updateGroupToken(g, groupId, tokenId, masterValue)
      );
      handleThemeTokenChange(updatedTokens);
    }, [activeThemeId, activeThemeTokens, handleThemeTokenChange]);
    ```

    Add a helper function `updateGroupToken` (pure, can be defined outside the component):
    ```typescript
    function updateGroupToken(group: TokenGroup, targetGroupId: string, tokenId: string, value: string): TokenGroup {
      if (group.id === targetGroupId) {
        return {
          ...group,
          tokens: group.tokens.map(t => t.id === tokenId ? { ...t, value } : t),
        };
      }
      return {
        ...group,
        children: group.children?.map(child => updateGroupToken(child, targetGroupId, tokenId, value)),
      };
    }
    ```

    **5. Determine which token data source to pass to TokenGeneratorForm:**
    When `activeThemeId !== null` and `activeGroupState === 'enabled'`, the form should display and allow editing of `activeThemeTokens`. The current `collectionToLoad` prop drives initial load. After load, edits go through `handleThemeTokenChange` (not the master `handleTokensChange`).

    The cleanest approach: add two new optional props to `TokenGeneratorForm` for theme mode:
    - `themeTokens?: TokenGroup[]` — when provided and non-empty, overrides the internal token groups for the selected group display
    - `onThemeTokensChange?: (tokens: TokenGroup[]) => void` — replaces `onTokensChange` in theme mode

    Pass these from `tokens/page.tsx`:
    ```tsx
    <TokenGeneratorForm
      ...existing props...
      themeTokens={activeThemeId ? activeThemeTokens : undefined}
      onThemeTokensChange={activeThemeId ? handleThemeTokenChange : undefined}
      isReadOnly={isThemeReadOnly}
      findMasterValue={activeThemeId ? findMasterValue : undefined}
      onResetToDefault={activeThemeId && !isThemeReadOnly ? handleResetToDefault : undefined}
    />
    ```

    **6. Extend TokenGeneratorForm to accept the new props:**
    Add to `TokenGeneratorFormProps`:
    ```typescript
    themeTokens?: TokenGroup[];
    onThemeTokensChange?: (tokens: TokenGroup[]) => void;
    isReadOnly?: boolean;
    findMasterValue?: (groupId: string, tokenPath: string) => string | undefined;
    onResetToDefault?: (groupId: string, tokenId: string, masterValue: string) => void;
    ```

    In `TokenGeneratorForm`, when `themeTokens` is provided and non-empty, use them as the active token groups for display instead of the internal `tokenGroups` state. Changes to tokens in theme mode should call `onThemeTokensChange` (not `onTokensChange`). The internal `tokenGroups` state continues to manage the master collection — theme tokens are an overlay, not a replacement.

    Pass `isReadOnly` down to each `TokenTableRow` via the group/section rendering.
    Pass `masterValue={findMasterValue?.(group.id, token.path)}` to each `TokenTableRow`.
    Pass `onResetToDefault` to each `TokenTableRow`.

    **Anti-pattern to avoid:** Do NOT modify `tokenGroups` state when `themeTokens` is active. They are independent state. The internal form state (add/delete tokens, mutations) must be guarded by `!isReadOnly` and `!themeTokens` to avoid cross-contamination.
