Create `src/app/api/collections/[id]/themes/route.ts` as a Next.js App Router route file.

**GET /api/collections/[id]/themes**
- Call `getRepository().findById(id)` — return 404 if not found
- Return `{ themes: doc.themes ?? [] }`

**POST /api/collections/[id]/themes**
- Body: `{ name: string }`
- Validate: name must be a non-empty string; return 400 if missing
- Load the collection to read existing groups from `doc.tokens` — extract top-level group keys (keys of `doc.tokens` that are objects, not `$`-prefixed)
- Determine default state per the CONTEXT.md decisions:
  - If `doc.themes.length === 0` (this is the first theme): all existing groups default to `'enabled'`
  - Otherwise: all existing groups default to `'disabled'`
- Build the new theme object:
  ```ts
  const theme: ITheme = {
    id: crypto.randomUUID(),
    name: body.name.trim(),
    groups: Object.fromEntries(groupIds.map(gid => [gid, defaultState])),
  };
  ```
- Use `TokenCollection.findByIdAndUpdate(id, { $push: { themes: theme } }, { new: true })` (import `TokenCollection` model and `dbConnect` directly — this route needs Mongo)
- Return `{ theme }` with status 201

**PUT /api/collections/[id]/themes/[themeId]** — handled in a SEPARATE file below (see note)

**DELETE /api/collections/[id]/themes/[themeId]** — also separate

**IMPORTANT:** For the PUT and DELETE routes that target a specific theme, create a second route file: `src/app/api/collections/[id]/themes/[themeId]/route.ts`

In that file:
- **PUT**: Body `{ name?: string; groups?: Record<string, ThemeGroupState> }`. Use `TokenCollection.findOneAndUpdate({ _id: id, 'themes.id': themeId }, { $set: { 'themes.$.name': body.name, 'themes.$.groups': body.groups } }, { new: true })` — only set fields that are present in body (check each separately). Return `{ theme: updatedTheme }` or 404.
- **DELETE**: `TokenCollection.findByIdAndUpdate(id, { $pull: { themes: { id: themeId } } })`. Return `{ success: true }` or 404.

Import `getRepository` from `@/lib/db/get-repository` for GET. Import `TokenCollection` + `dbConnect` directly for the array mutation operations (POST, PUT, DELETE) since `ICollectionRepository` doesn't expose array-push semantics.
