**Note on NextRequest vs Request parameter types:** Some of these handlers declare `request: NextRequest` (from `next/server`) instead of `request: Request`. The requireAuth() function takes no parameters — it calls `getServerSession(authOptions)` internally, which reads the session cookie from the request context automatically in App Router. The guard pattern is identical regardless of whether the handler uses `NextRequest` or `Request`.
