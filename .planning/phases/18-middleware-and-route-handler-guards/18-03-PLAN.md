  <how-to-verify>
    Start the dev server: `yarn dev` from the project root.

    **Scenario 1 — Unauthenticated page redirect (AUTH-02):**
    Open a new browser window in incognito mode (no cookies). Visit http://localhost:3000/collections.
    Expected: Browser redirects to http://localhost:3000/auth/sign-in?callbackUrl=%2Fcollections
    The callbackUrl query param must be present in the URL.

    **Scenario 2 — API write returns 401 without session (ARCH-02):**
    In a terminal (no session cookie), run:
    ```
    curl -s -X POST http://localhost:3000/api/collections \
      -H "Content-Type: application/json" \
      -d '{"name":"test"}'
    ```
    Expected output: `{"error":"Unauthorized"}`

    Also verify the HTTP status code:
    ```
    curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:3000/api/collections \
      -H "Content-Type: application/json" \
      -d '{"name":"test"}'
    ```
    Expected output: `401`

    **Scenario 3 — CVE-2025-29927 exploit non-functional:**
    ```
    curl -v -H "x-middleware-subrequest: middleware" http://localhost:3000/collections 2>&1 | grep -E "< HTTP|Location:"
    ```
    Expected: HTTP 307 (or 302/303) redirect with Location: pointing to /auth/sign-in.
    NOT expected: HTTP 200 OK with collections page HTML (that would indicate the CVE is open).

    **Scenario 4 — Authenticated user redirected away from sign-in page:**
    Sign in at http://localhost:3000/auth/sign-in with valid credentials (SUPER_ADMIN_EMAIL + password from .env.local).
    After sign-in, while still authenticated, navigate directly to http://localhost:3000/auth/sign-in.
    Expected: Immediate redirect to http://localhost:3000/collections — sign-in form is not shown to authenticated users.
