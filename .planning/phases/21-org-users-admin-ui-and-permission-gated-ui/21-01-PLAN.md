Replace the existing `async jwt({ token, user })` callback with the extended version that re-fetches role and status from DB every 60 seconds.

The current callback (lines 39-51) sets token fields only at initial sign-in (when `user` is present) and applies the SUPER_ADMIN_EMAIL override. Replace it entirely with:

```typescript
async jwt({ token, user }) {
  // Initial sign-in: set all fields and seed roleLastFetched
  if (user) {
    token.id   = (user as { id: string }).id;
    token.role = (user as { role: string }).role;
    token.name = (user as { name: string }).name;
    token.roleLastFetched = Date.now();
  }

  // SUPER_ADMIN_EMAIL short-circuit — apply first, return early to skip DB re-fetch
  // Checks token.email (always present) per AUTH-06 decision in STATE.md
  if (token.email === process.env.SUPER_ADMIN_EMAIL) {
    token.role = 'Admin';
    token.roleLastFetched = Date.now(); // keep timestamp fresh, no DB hit needed
    return token;
  }

  // Role re-fetch when stale (> 60 seconds)
  // token.id is absent in very old pre-v1.5 sessions — skip re-fetch safely
  if (!token.id) return token;

  const ROLE_TTL_MS = 60 * 1000;
  const lastFetched = (token.roleLastFetched ?? 0) as number;
  if (Date.now() - lastFetched > ROLE_TTL_MS) {
    await dbConnect();
    const dbUser = await User.findById(token.id as string)
      .select('role status')
      .lean() as { role: string; status: string } | null;

    if (!dbUser || dbUser.status === 'disabled') {
      // User deleted or removed — invalidate session by returning empty token
      // next-auth v4: returning {} forces re-sign-in on the client
      return {} as typeof token;
    }

    token.role = dbUser.role;
    token.roleLastFetched = Date.now();
  }

  return token;
},
```

Key implementation notes (from RESEARCH.md):
- `user` is only present on initial sign-in, NOT on subsequent JWT refreshes — do not rely on it for re-fetch
- SUPER_ADMIN_EMAIL check comes BEFORE the DB re-fetch block and returns early — prevents any DB override of the superadmin's role
- Returning `{}` (empty object) is the correct next-auth v4 mechanism to invalidate a session; do NOT return `null` (it is ignored)
- `token.id` guard handles pre-Phase 16 sessions that lack the `id` field — skip re-fetch rather than crash

The `dbConnect` and `User` imports are already at the top of the file (lines 9-10). No new imports needed.

After replacing the jwt callback, verify the session callback (lines 53-59) is unchanged — it still reads `token.id` and `token.role` to populate `session.user`.
