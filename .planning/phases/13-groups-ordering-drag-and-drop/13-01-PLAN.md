    **Step 1 — Install packages:**

    **Step 2 — Create `src/utils/groupMove.ts` with the following exports:**

    **`FlatNode` interface:**
    ```ts
    export interface FlatNode {
      group: TokenGroup;
      depth: number;
      parentId: string | null;
      index: number;       // index within sibling array
      displayLabel: string;
    }
    ```

    **`flattenTree(groups, parentId?, depth?)` — recursive flatten:**
    - For each group, compute `displayLabel` using `parseGroupPath(group.name)[last segment]`
    - Push `{ group, depth, parentId, index, displayLabel }`
    - Recurse into `group.children` with `parentId = group.id`, `depth + 1`
    - Return `FlatNode[]`

    **`buildTreeFromFlat(flat)` — rebuild nested tree from flat array:**
    - Identify roots: nodes where `parentId === null`
    - For each non-root, attach to parent's `children` array
    - Sort children by their `index` field (preserves drag order within each level)
    - Return `TokenGroup[]`

    **`applyGroupMove(groups, activeId, overId, themes?)` — main cascade:**
    This is the critical function. Follow this exact cascade sequence:

    1. Call `flattenTree(groups)` to get `flatNodes`.
    2. Find `activeNode` (node with `group.id === activeId`) and `overNode` (node with `group.id === overId`).
    3. If not found or `activeId === overId`, return `{ groups, themes: themes ?? [] }` unchanged.
    4. Compute `newParentId`: use `overNode.parentId` (insert as sibling of over, at same level). This implements "insert at same level as the hovered item" — the simple heuristic from research.
    5. Compute `newIndex`: use the over item's flat index position to determine insertion order.
    6. Remove `activeNode` from `flatNodes` (splice by index).
    7. Re-insert `activeNode` at the over item's position (before it as sibling).
    8. Recompute `parentId` and `index` for the moved node and all affected siblings.
    9. Call `buildTreeFromFlat(updatedFlat)` to get `newGroups` (tree without ID rewrite yet).
    10. Determine if reparenting occurred: `activeNode.parentId !== newParentId`.
    11. **If reparenting (new parent differs):**
        a. Compute `oldSlashPrefix` = `activeNode.group.id` (e.g. `"brand/primary"`)
        b. Compute `newSlashPrefix` = `newParentId ? newParentId + '/' + leafName : leafName` where `leafName` is the last segment of the moved group's name (from `parseGroupPath`).
        c. Check for name collision in the new parent's children. If a sibling already has the same leaf name, append `-2`, `-3`, etc. (increment until no collision, up to `-10`).
        d. Call `rewriteSubtreeIds(movedGroup, oldSlashPrefix, newSlashPrefix)` — returns the moved group with all descendant IDs, paths, parents, and token IDs rewritten.
        e. Call `rewriteAliasesInTree(newGroups, oldDotPrefix, newDotPrefix)` — scans ALL groups for alias token values `{old.dot.path.*}` and rewrites them. `oldDotPrefix = oldSlashPrefix.replaceAll('/', '.')`, `newDotPrefix = newSlashPrefix.replaceAll('/', '.')`.
        f. If `themes` provided, for each theme:
           - Apply same `rewriteSubtreeIds` to the theme's tokens snapshot
           - Apply same `rewriteAliasesInTree` to the theme's tokens snapshot
           - Rename keys in `theme.groups` Record: for each key starting with `oldSlashPrefix`, replace prefix with `newSlashPrefix` using `migrateThemeGroupKeys(theme, oldSlashPrefix, newSlashPrefix)`
        g. Return `{ groups: updatedGroups, themes: updatedThemes }`.
    12. **If sibling reorder only (no reparenting):** No ID rewrite needed. Sync theme snapshots by applying the same array ordering to each theme's `.tokens` (reorder children arrays to match master). Return `{ groups: newGroups, themes: syncedThemes }`.

    **Private helpers inside groupMove.ts:**

    `rewriteSubtreeIds(group, oldSlashPrefix, newSlashPrefix)`:
    - Replace all occurrences of `oldSlashPrefix` with `newSlashPrefix` in: `group.id`, `group.path`, `group.parent`
    - Rewrite all `group.tokens` IDs: `t.id.replace(oldSlashPrefix, newSlashPrefix)`
    - Recurse into `group.children`
    - Return new `TokenGroup`

    `rewriteAliasesInTree(groups, oldDotPrefix, newDotPrefix)`:
    - For each group recursively, for each token whose `value` is a string starting with `{`:
      - Replace occurrences of `{${oldDotPrefix}.` with `{${newDotPrefix}.` using a regex with `escapeRegex` helper
    - Return new `TokenGroup[]`

    `migrateThemeGroupKeys(theme, oldSlashPrefix, newSlashPrefix)`:
    - Iterate `Object.entries(theme.groups)`, rename any key starting with `oldSlashPrefix`
    - Return new `ITheme` with updated `groups` Record

    `syncThemeTokenOrder(themeTokens, masterGroups)`:
    - Apply the same child ordering from `masterGroups` to `themeTokens` (match by group ID)
    - Used for sibling reorder case to keep theme snapshots in sync
    - Returns new `TokenGroup[]` for the theme

    **Import from `@/utils`:** `parseGroupPath`
    **Import types:** `TokenGroup` from `@/types/token.types`, `ITheme` from `@/types/theme.types`
    **Export:** `FlatNode`, `flattenTree`, `buildTreeFromFlat`, `applyGroupMove`

    Do NOT use any React hooks or Next.js imports in this file — it must remain a pure util.
