  <action>
Read each file in full before editing.

For each component, apply this migration pattern:

**Dialog structure replacement:**
- Remove custom overlay div (typically `fixed inset-0 bg-black/50 z-50 flex items-center justify-center` or similar backdrop)
- Remove custom modal container div
- Wrap content with:
  ```tsx
  import { Dialog, DialogContent, DialogHeader, DialogTitle, DialogFooter, DialogDescription } from '@/components/ui/dialog';
  import { Button } from '@/components/ui/button';
  import { Input } from '@/components/ui/input';

  <Dialog open={props.open} onOpenChange={(open) => { if (!open) props.onClose(); }}>
    <DialogContent>
      <DialogHeader>
        <DialogTitle>{title}</DialogTitle>
        [<DialogDescription>...</DialogDescription> if there is a subtitle/description]
      </DialogHeader>
      {/* body content */}
      <DialogFooter>
        {/* action buttons */}
      </DialogFooter>
    </DialogContent>
  </Dialog>
  ```

**Props interface:** Each component has an `open: boolean` prop (or a show/visible boolean) and an `onClose` callback. Wire `open` to `Dialog open` prop and `onClose` to `onOpenChange`. If the component currently shows/hides via conditional render in parent (not via `open` prop), keep that pattern — just pass `open={true}` when the dialog is rendered (since parent already conditionally mounts it).

**Button migration:** Replace all `<button>` with shadcn `<Button>`. Destructive actions → `variant="destructive"`. Cancel/close → `variant="outline"`. Primary confirm → `variant="default"`.

**Input migration:** Replace `<input type="text">` with `<Input>`. Preserve all props.

**BuildTokensModal specifics:** Has a multi-step flow (build → results). Preserve the step-based conditional rendering inside DialogContent.

**JsonPreviewDialog specifics:** Has a `<pre>` / `<code>` block for JSON display — keep that unchanged, just wrap with Dialog.

**LoadCollectionDialog specifics:** Likely has a list of collections to select from — keep list rendering, wrap with Dialog.

Do NOT change any business logic, callback functions, or data props.
