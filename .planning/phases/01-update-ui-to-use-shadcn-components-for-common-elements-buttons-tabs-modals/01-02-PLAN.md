Read src/app/page.tsx in full before editing.

The page currently has a custom tab switcher using plain `<button>` elements with conditional Tailwind classes (active/inactive states). Replace with shadcn Tabs:

1. Add imports at top of file:
   ```typescript
   import { Tabs, TabsList, TabsTrigger, TabsContent } from '@/components/ui/tabs';
   import { Button } from '@/components/ui/button';
   ```

2. Replace the tab switcher JSX. The existing pattern (around line 442+) has two `<button>` elements that call `switchTab('view')` and `switchTab('generate')`. Replace this with:
   ```tsx
   <Tabs value={activeTab} onValueChange={(v) => switchTab(v as 'view' | 'generate')}>
     <TabsList>
       <TabsTrigger value="view">View</TabsTrigger>
       <TabsTrigger value="generate">Generate</TabsTrigger>
     </TabsList>
   </Tabs>
   ```
   Keep the existing `{/* Tab content */}` divs (with `hidden` class logic) below — those are controlled by `activeTab` state, not by shadcn TabsContent. Do NOT wrap them in TabsContent (the state management preserves form state across tab switches by always mounting both divs).

3. Replace all plain `<button>` elements in page.tsx with shadcn `<Button>`. Map variants:
   - Primary/action buttons (dark bg, e.g. "Import from Figma", "Export to Figma", "Load Tokens" buttons) → `variant="default"`
   - Secondary/cancel/outline buttons → `variant="outline"`
   - Destructive/delete actions → `variant="destructive"`
   - Icon-only buttons → `variant="ghost" size="icon"`

   Preserve all existing `onClick` handlers, `disabled` conditions, and conditional rendering exactly.

4. Do NOT change any state management, routing logic (`switchTab`, `router.push`), or data fetching.
