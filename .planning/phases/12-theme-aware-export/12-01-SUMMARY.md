# Phase 12 Plan 01: Theme Token Merge Foundation Summary

**One-liner:** Pure `mergeThemeTokens` helper blending master collection tokens with theme group states (disabled/source/enabled), plus `themeLabel` comment-header injection in the build-tokens route for CSS/SCSS/LESS/JS/TS outputs.

## What Was Built

### Task 1: themeTokenMerge.ts + BuildTokensRequest extension

Created `/Users/user/Dev/atui-tokens-manager/src/lib/themeTokenMerge.ts` — a pure function that:
- Returns `masterTokens` unchanged when `theme === null` (Collection default pass-through)
- Parses master tokens via `tokenService.processImportedTokens` to get a `TokenGroup[]` tree
- Builds a Map from `theme.tokens` (already `TokenGroup[]` — NOT re-parsed)
- Walks root-level master groups and applies theme state logic:
  - `disabled` → excluded entirely from merged output
  - `source` → uses master collection values
  - `enabled` → uses theme snapshot, falls back to master if group not found
- Returns `tokenService.generateStyleDictionaryOutput(mergedGroups, namespace)` — SD-ready JSON

Extended `BuildTokensRequest` in `/Users/user/Dev/atui-tokens-manager/src/types/token.types.ts`:
- Added `themeLabel?: string` — optional field for downstream comment injection

### Task 2: Comment header injection in build-tokens route

Updated `/Users/user/Dev/atui-tokens-manager/src/app/api/build-tokens/route.ts`:
- Added `COMMENT_FORMATS = new Set(['css', 'scss', 'less', 'js', 'ts'])` — JSON excluded
- After `buildTokens()` call, if `body.themeLabel` is present:
  - Constructs `/* Generated by Tokens Manager, theme: [themeLabel] */\n`
  - Prepends header to `brandOutput.content` for every matching format
- `style-dictionary.service.ts` is NOT modified — it remains pure
- Typed `result` explicitly as `BuildTokensResult`
