# Phase 12: Theme-Aware Export - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a theme selector to the Config page export controls so Style Dictionary and Figma Variables exports are targeted at a specific theme's token values (or the collection default). Figma Variables export gains multi-mode support — one mode per enabled theme.

GitHub sync (publish to live SD-based repo) is a separate flow and is NOT in scope for this phase.

</domain>

<decisions>
## Implementation Decisions

### Theme selector placement
- One shared selector above the export buttons — applies to both SD and Figma exports
- Label: "Export theme:" — clearly scoped to export, above the action buttons
- Default: always "Collection default" on page load (not persisted)
- "Collection default" is a pass-through: the existing SD export code runs unchanged when selected (no new merge logic)

### SD export — group state handling
- **Disabled groups**: excluded from export entirely (not included in the SD build)
- **Source groups**: included using collection-default token values (consistent with Phase 11 semantics)
- **Enabled groups**: use the selected theme's embedded token values

### SD export — output format
- Filename and folder structure: same as today (no theme name in ZIP or file names)
- Comment header in code format files (CSS, SCSS, LESS, JS, TS): `/* Generated by Tokens Manager, theme: [theme name or "Collection default"] */`
- JSON format: no comment (JSON does not support comments; JSON stays clean)

### Figma Variables — mode structure
- "Default" mode always included as the first mode, using collection-default token values
- Each enabled theme becomes a subsequent mode using that theme's embedded token values
- Source groups in each theme's mode: use collection-default values (same as SD export)
- No enabled themes: export Default mode only (graceful fallback, no error)
- Mode naming: theme name as-is (the theme's `name` field from the database)

### Figma Variables — export scope
- Figma export always includes ALL enabled themes as modes — it ignores the theme selector entirely
- The theme selector only affects the SD export
- Rationale: Figma Variables are designed to hold all modes in one export

### Figma Enterprise gate UX
- Inline note below the Figma Export button: "Figma Variables API requires Enterprise plan."
- Button remains active — user can attempt the export; the API returns an error if they lack access
- Error handling: toast notification showing the API error message (consistent with existing app patterns)

### Claude's Discretion
- Exact wording of the "Export theme:" label and selector styling
- Selector component type (shadcn Select, same pattern as theme selector on Tokens page)
- Order and visual grouping of the theme selector relative to existing Config page content

</decisions>

<specifics>
## Specific Ideas

- Comment format for code files: `/* Generated by Tokens Manager, theme: [theme name] */` — similar to generated-file header conventions
- The Figma Enterprise note should not be alarming — it's informational, not a block

</specifics>
