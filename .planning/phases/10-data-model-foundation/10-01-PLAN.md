**3. src/app/api/collections/[id]/themes/route.ts**

Two changes in the POST handler:

a) **10-theme count guard** — insert BEFORE the `theme` object is constructed (after `existingThemes` is assigned on line 61). Remove the `void existingThemes` suppression line and replace the block with:

```typescript
const existingThemes = (collection.themes as ITheme[]) ?? [];

if (existingThemes.length >= 10) {
  return NextResponse.json(
    { error: 'Maximum 10 themes per collection' },
    { status: 422 }
  );
}
```

b) **Embed groupTree as theme.tokens** — update the `theme` object literal (currently lines 65-69) to add the `tokens` field. Store `groupTree` (the full hierarchical tree returned by `processImportedTokens`), NOT `flattenAllGroups(groupTree)` which is only the flat list used for group IDs:

```typescript
const theme: ITheme = {
  id: crypto.randomUUID(),
  name: body.name.trim(),
  groups: Object.fromEntries(groupIds.map((gid) => [gid, defaultState])),
  tokens: groupTree,  // full tree snapshot — not the flat list
};
```

**CRITICAL pitfalls to avoid:**
- Do NOT store `flattenAllGroups(groupTree)` as `tokens` — that is a flat array with no `children`. Store `groupTree` (the return value of `processImportedTokens(...).groups`).
- Do NOT use positional `$set: { 'themes.$.tokens': ... }` anywhere — Mongoose Mixed-array bug. The whole-array `$push: { themes: theme }` already used here is correct.
- Do NOT use HTTP 400 for the count cap — the locked decision is 422 Unprocessable Entity.
- The type change makes `tokens` required; the only `ITheme` construction site is in this file, so the edit is self-contained.
