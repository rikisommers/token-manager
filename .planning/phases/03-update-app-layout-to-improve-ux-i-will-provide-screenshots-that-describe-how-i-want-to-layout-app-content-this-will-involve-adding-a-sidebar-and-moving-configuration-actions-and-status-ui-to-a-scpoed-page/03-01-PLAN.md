  <name>Task 1: Create AppSidebar component</name>
  <files>src/components/AppSidebar.tsx</files>
  <action>
Create `src/components/AppSidebar.tsx` as a 'use client' component. The sidebar is fixed, full-height, ~200px wide, and renders as a dark-toned column (reference: Tokens Studio sidebar — dark background, white/light text).

Structure from top to bottom:
1. App name — "ATUI Tokens" in white text, semibold, at the top of the sidebar (padded ~16px)
2. Collection selector — render a `<CollectionSelectorSidebar />` inline sub-component (defined in the same file) that wraps the existing `CollectionSelector` component. Pass `collections`, `selectedId`, `loading`, and `onChange` as props received from AppSidebar's own props. Style it to fit the dark sidebar (use a compact Select with a dark-friendly trigger style).
3. Nav items — three items: Tokens (`/`), Configuration (`/configuration`), Settings (`/settings`). Each item is a Next.js `<Link>` with an icon and label side by side. Use Lucide React icons (already available via shadcn): `Palette` for Tokens, `Settings2` for Configuration, `SlidersHorizontal` for Settings. Active item (detected via `usePathname()`) gets a filled dark background: `bg-white/15` with `text-white`, inactive items get `text-white/70 hover:bg-white/10`.

Sidebar background: `bg-gray-900`. Nav section starts after the collection selector with some vertical spacing. Nav items: full-width, px-3 py-2, rounded-md, flex items-center gap-3, text-sm font-medium.

AppSidebar props interface:
```typescript
interface AppSidebarProps {
  collections: { _id: string; name: string }[];
  selectedId: string;
  loading: boolean;
  onSelectionChange: (id: string) => void;
}
```

The component must be exported as `export function AppSidebar(...)`.

Import `CollectionSelector` from `@/components/CollectionSelector` — do not duplicate the select logic.

Note: The sidebar does NOT manage collection state itself — it receives state as props from the layout or a client wrapper. Since `layout.tsx` is a server component, the sidebar state (collections list, selectedId) must be managed in a client wrapper. Create a `SidebarWrapper.tsx` client component in the same file (or a separate file `src/components/SidebarWrapper.tsx`) that holds the collections state via `useState`/`useEffect` fetching `/api/collections`, then renders `AppSidebar` with the fetched data. This approach keeps the root layout.tsx server-side while the sidebar manages its own collection state independently of the page content.

Actually: to keep things simple, put the state logic directly into AppSidebar (make it fully self-contained — fetches `/api/collections` itself via useEffect, manages `selectedId` via localStorage). It stores `selectedId` in localStorage key `atui-selected-collection-id`. When `onSelectionChange` is called externally (prop is optional: `onSelectionChange?: (id: string) => void`), forward the call. Pages that need to react to collection changes should use their own state (or URL params).

Final AppSidebar: self-contained, fetches collections itself, reads/writes localStorage for selectedId, renders the sidebar UI. Export it.
