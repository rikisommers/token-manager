  <action>
Create `src/app/collections/[id]/settings/page.tsx` as a 'use client' component.

Accept params: `{ params: { id: string } }`

State:
- `collectionName: string` (for heading)
- `figmaToken: string` (controlled input)
- `figmaFileId: string` (controlled input)
- `githubRepo: string` (controlled input)
- `githubBranch: string` (controlled input)
- `saveStatus: 'idle' | 'saving' | 'saved' | 'error'`
- `loading: boolean` (initial fetch)

On mount (useEffect):
1. Fetch GET /api/collections/[id] to get collection name + current config values
2. For each of the four config fields: if the DB value is non-null, use it; otherwise pre-populate from localStorage:
   - figmaToken: localStorage.getItem('atui-figma-token') ?? ''
   - figmaFileId: localStorage.getItem('atui-figma-file-id') ?? ''
   - githubRepo: localStorage.getItem('atui-github-repo') ?? ''
   - githubBranch: localStorage.getItem('atui-github-branch') ?? ''
   (Note: use these localStorage keys — they are the keys that FigmaConfig/GitHubConfig currently use. Verify actual key names by checking FigmaConfig.tsx and GitHubConfig.tsx before hardcoding. If different keys are used, adopt the actual keys.)

Auto-save implementation:
- Create a `useEffect` that watches all four field values
- Use `useRef` to hold a debounce timer: `const debounceRef = useRef<ReturnType<typeof setTimeout> | null>(null)`
- On any field change: clear existing timer, set `saveStatus = 'saving'`, set new timer (800ms)
- Timer callback: PUT /api/collections/[id] with `{ figmaToken: v || null, figmaFileId: v || null, githubRepo: v || null, githubBranch: v || null }` (empty string → null)
- On success: set `saveStatus = 'saved'`, then after 2000ms set back to 'idle'
- On error: set `saveStatus = 'error'`
- Do NOT auto-save on the initial mount load (use a `didMountRef` ref to skip first run)

Clear/Reset action:
- "Clear all" button in each section OR one global "Reset" button
- Implement as per-section: a small "Clear" link/button next to each section heading that sets that section's fields to '' and immediately saves (null) to DB — skip the debounce, call save immediately
- Button style: `text-xs text-gray-400 hover:text-red-500 ml-2`

Save status indicator:
- Show in the page header area: a small badge
- 'saving': spinning icon + "Saving..."
- 'saved': green checkmark + "Saved" (fades after 2s)
- 'error': red dot + "Error saving"
- Use inline Tailwind; no toast library needed here

Page layout:
```
div className="p-6 max-w-2xl"
  [heading row]
    h1 "Settings: {collectionName}" + saveStatus badge

  [Figma section]
    h2 "Figma" + "Clear" button
    label + input for "Figma Token" (type="password" — mask the token)
    label + input for "Figma File ID"
    small help text below each field

  [GitHub section]
    h2 "GitHub" + "Clear" button
    label + input for "GitHub Repo" (format: "org/repo")
    label + input for "GitHub Branch"
    small help text below each field
```

Use shadcn Input component (`@/components/ui/input`) and shadcn Label (`@/components/ui/label`) if available, otherwise standard Tailwind-styled inputs.

Do NOT import FigmaConfig or GitHubConfig components — they have modal/toggle behavior that conflicts with the always-visible auto-save pattern. Build the fields directly in this page.

Do NOT add any "Test connection" or "Validate" functionality — that is out of scope for Phase 4.
